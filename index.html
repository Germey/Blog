API 开发教程 在 Jan 中使用 Ace Data Cloud

Jan 是一款主打「离线优先 + 隐私保护」的开源 AI 桌面客户端。它自带本地推理引擎（基于 llama.cpp，Apple Silicon 上可用 MLX），可以直接在本机运行开源模型；同时通过 Remote Engine 机制支持任意 OpenAI 兼容端点。借助这一能力可以接入 Ace Data Cloud，让本地模型与云端 Claude、GPT、Gemini、Grok、DeepSeek、Kimi、GLM 等 60+ 大模型共存于同一客户端，一份 API Token 即可使用。本文介绍配置流程。

申请流程

要在 Jan 中接入 Ace Data Cloud，首先到 Ace Data Cloud 控制台，获取您的 API Token，留作备用。

如果你尚未登录或注册，会自动跳转到登录页面邀请您来注册和登录，登录注册之后会自动返回当前页面。

在首次申请时会有免费额度赠送，可以免费体验 Ace Data Cloud 的模型服务。

下载 Jan

到 Jan 官网 按平台下载（Windows、macOS、Linux）。Apple Silicon 机型请下载 arm64 包以获得最佳性能。

配置 Ace Data Cloud

Jan 把模型分为 Local Engine（本机运行）与 Remote Engine（第三方 API）两类。Ace Data Cloud 走 OpenAI 兼容协议，按 Remote Engine 接入。

打开 Jan，进入 Settings → Model Providers，找到 OpenAI 卡片并编辑：

Base URL https://api.acedata.cloud/v1 必须以 /v1 结尾，结尾不要带斜杠

Base URL 必须以 /v1 结尾是 Jan 官方文档的明确约定。如果不想覆盖原 OpenAI 配置，可以「+ Add Provider」新建一个 openai-compatible 类型的供应商，填入相同内容。

API 开发教程 AI Chat 定时任务说明

定时任务（/aichat2/scheduled-tasks）让你的 AI Agent 在无人值守的情况下按计划自动运行——每天汇总新闻、每周生成报告、每小时监控数据等任务，全部交给平台在后台自动完成，完成后推送通知让你直接看结果。

功能简介

  • 全能力 agentic 执行：定时任务复用 AI Chat v2 的完整工具链（网页搜索、读取文件、调用 MCP 服务器等），不仅仅是普通问答。
  • 灵活的调度规则：支持每天（daily）、每小时（hourly）、每周（weekly）以及自定义 Cron 表达式。
  • 推送通知：任务完成后通过 Web Push / FCM / APNs 推送到你的浏览器和手机，点击即可跳转到结果对话。
  • 模板变量：在提示词中使用 1784754045000（今日日期）、（第几次运行）、（上次输出摘要）实现动态提示。
  • 透明计费：每次运行按实际使用的模型和 token 数计费，与手动发起的对话价格完全相同。在控制台「用量」页可看到以「Scheduled Task」为来源的独立记录。

申请方式

同 AI Chat v2，在 Ace Data Cloud 控制台 申请 aichat API 权限，获取 API Token 即可同时使用对话和定时任务功能。

通过 Nexior 使用

打开 Nexior，在左侧对话列表顶部点击「定时任务」（时钟图标）进入管理页。

  1. 点击「新建」，填写任务名称、提示词、模型和执行频率。
  2. 保存后任务立即生效，下一次调度时间按你设定的频率自动计算。
  3. 点击任一任务卡片可查看历次运行记录（时间、状态、结果链接）。
  4. 随时通过开关暂停 / 恢复，或点击删除按钮永久移除。

通过 API 使用

所有操作统一走 POST /aichat2/scheduled-tasks，用 action 字段区分。

注意事项

  • 无人值守模式：定时任务运行时如果 AI 需要调用 ask_user_question（请求你确认）或需要第三方授权（request_user_consent），该工具会被自动禁用，任务可能跳过相关步骤。如果你的用例需要这类交互，建议先手动发起一次对话完成授权，再创建定时任务。
  • 余额：每次运行前会检查账户余额。余额不足时该次运行会标记为失败，并通过推送通知提醒你充值。连续 5 次失败后任务自动暂停。
  • ends_at 默认值：未设置结束时间时，任务默认在创建后 3 个月自动停止，防止遗忘的僵尸任务持续扣费。可在创建时通过 ends_at（Unix 时间戳）延长。
  • 最大运行时长：单次运行最长 15 分钟。

计费

定时任务与普通 AI Chat v2 使用完全相同的价格体系，按模型和 token 数计费。在控制台用量页，定时任务产生的记录会以「AceChat Scheduled Run」独立展示，便于区分手动对话和自动运行的消耗。

Safety defaults

  • EMAIL_MARKETING_ALLOW_REAL_RECIPIENTS defaults to false.

Seed rollout

  1. Keep EMAIL_MARKETING_ALLOW_REAL_RECIPIENTS=false.
  2. Keep exactly the owner-approved seed addresses in EMAIL_MARKETING_TEST_RECIPIENTS.
  3. Refresh audience profiles and create a campaign using a narrow preset.

Recovery

  • Pause globally first by setting EmailDeliveryControl.sending_enabled=false in the admin console.
  • reconcile_email_delivery recovers expired leases and never retries a provider request with an ambiguous outcome.
  • Ambiguous sends become OutcomeUnknown after the reconciliation window and require manual review.
  • Provider/user suppressions cannot be manually deactivated; only ManualSafety suppressions can be removed with an audit reason.
  • Recipient PII is purged after the configured retention period; hashes and aggregate delivery records remain for safety and reporting.

技术教程 OpenCode 对接 Seedance MCP

📘 Overview：OpenCode MCP Overview →

Seedance 在“短视频 / 竖屏素材 / 动作演绎”上的表现尤其稳定，可以一次性给出带音频的成片，非常适合直接用作社交媒体素材。在 OpenCode 里接上 Seedance MCP Server 后，可以从需求一句话直接出动效视频，省去剪辑、配音的中间步骤。

Seedance MCP 适合做 Reels / 抖音 / 小红书短视频、产品 demo 动效、技术分享开场片头等“短而有冲击力”的视频场景。

获取 API Token

使用 Seedance MCP Server 之前，需要先准备一个 Ace Data Cloud API Token。OpenCode 和 Claude Code 共用一套 Token，获取流程一样：

  1. 打开 Ace Data Cloud 控制台 - 应用列表，获取您的 API Token，留作备用。
  2. 如果你尚未登录或注册，会自动跳转到登录页面；登录注册之后会自动返回当前页面。
  3. 首次申请时会有免费额度赠送，可以先免费体验 Seedance MCP 服务。

一个 Token 可以使用 AceData Cloud 提供的全部 11 个 MCP Server，无需为 Seedance 单独申请。文档和截图里建议只展示脱敏形式，例如 3b78cc40dd3b43db806a4300....，不要把完整 Token 贴到公开仓库、Issue、截图或聊天记录里。

API 开发教程 自带密钥（BYOK）使用教程

BYOK（Bring Your Own Key）让你在 Nexior / Studio 的对话界面里，直接使用自己已有的上游服务商 API Key，完全绕开平台的账号池和积分计费。配置后该模型的对话请求会用你的 Key 直接打到 OpenAI / Anthropic / Google / xAI / DeepSeek / Moonshot / Zhipu —— 平台只做转发和工具拼装，不消耗任何积分。

适合场景：你已经有上游账号 + 余额，希望复用平台的 UI（云历史、Artifacts、引用、工具调用、Connector），但不想按平台的零售价付钱。

每个 Provider 可保存多把 Key（例如个人 Key + 公司 Key），并通过 Label 区分。

5 分钟实操

第 1 步：打开 BYOK 设置

登录 studio.acedata.cloud → 右上角头像 → Settings → Custom API Key。

第 2 步：添加一把 OpenAI Key

  1. 点 Add API Key；
  2. Provider 选 OpenAI；
  3. Label（可选）：My Work ID；
  4. API Key：粘贴你的 sk-...；
  5. Custom Base URL（可选）：留空走官方地址；如要走代理（例如 https://openai.acedata.cloud），把整段 base URL 填这里；
  6. 勾选 Active 后保存。

第 3 步：开始对话

回到聊天，模型选 gpt-5.5 / gpt-5-pro / gpt-4.1 任一 OpenAI 模型，发一条消息。Studio 会优先匹配你已激活的 BYOK Key，并在对话顶部显示 BYOK · OpenAI 徽章 —— 看到这个徽章说明这条对话没消耗平台积分。

如果同一 Provider 有多把 Key，Studio 会按最近使用 + Label 优先选；你也能在徽章上点击切换。

注意事项

  1. 只覆盖该 Provider 的模型：例如配置 OpenAI Key 只影响 GPT 系列，切到 Claude 仍会扣平台积分（除非你也配了 Anthropic Key）。
  2. 工具调用、引用、Artifacts、Connector 不受影响：这些都是平台侧的能力，不论是否走 BYOK 都正常工作。
  3. 错误处理透明：上游 401 / 429 / 5xx 会原样返回到对话，让你能直接看到上游 Key 的状态。
  4. Key 加密保存：平台只在转发请求时解密；不写日志、不会出现在客户端。
  5. 临时停用：把 Active 关掉而不删，下一条对话立即回退到平台积分计费。
  6. 不支持流式生效切换：对话进行中不会改用 BYOK，新建对话或刷新页面后生效。

常见问题

Q：自定义 Base URL 怎么配？
A：场景 1（境内代理）：填 https://openai.acedata.cloud/v1，请求会先走 AceData Cloud 边缘再到 OpenAI；场景 2（私有网关）：填你公司内部的 OpenAI 兼容网关。注意末尾不要带斜杠、不要带 /chat/completions。

Q：能跟订阅 / 充值并存吗？
A：能。BYOK 优先于积分计费；当 BYOK Key 失效（401）时不会自动 fallback 到积分，会直接报错让你修 Key。

Q：能管理团队成员的 Key 吗？
A：当前 BYOK 是个人维度的。团队 / 站长可以在 Subsite 维度统一配置上游账号池，请参考 Subsite 教程。

API 开发教程 Claude Code CC Switch 图形化配置教程

Claude Code 原生通过 ANTHROPIC_BASE_URL 与 ANTHROPIC_AUTH_TOKEN 两个环境变量决定请求发往哪里，把它们指向 Ace Data Cloud 即可用更低的价格、无需 Anthropic 官方订阅来使用 Claude Code。手动编辑 .claude/settings.json 对新手并不友好，而 CC Switch 提供了一个图形界面，帮你把这两个变量一键写好、并在多套供应商之间即时切换。

本文介绍如何用 CC Switch 把 Claude Code 接入 Ace Data Cloud。

什么是 CC Switch

CC Switch（官网 ccswitch.io）是一个开源、跨平台的桌面端配置管理器，用于统一管理 Claude Code、Claude Desktop、Codex、Gemini CLI、OpenCode、OpenClaw、Hermes 等多个 AI 编程工具的供应商配置。它基于 Tauri 2 构建，采用 MIT 协议开源，在 GitHub 上已获得超过 11 万 Star（参见 farion1231/cc-switch）。

它的核心价值在于（详见官方 User Manual）：

  • 免手动改配置：不用再手写 settings.json / config.toml / .env，图形界面填一次即可。
  • 一键切换：预存多套供应商，从主界面或系统托盘即时切换。
  • 原子写入：所有数据存于本地 SQLite 数据库（~/.cc-switch/cc-switch.db），采用「临时文件 + 重命名」的原子写入，避免配置损坏。

CC Switch 的工作原理是在底层动态覆盖 Claude Code 的 ANTHROPIC_BASE_URL 和 ANTHROPIC_AUTH_TOKEN，把原本发往 Anthropic 官方的请求重定向到你配置的第三方接口（本例即 Ace Data Cloud）。这一机制在社区的图文教程中也有详细说明。

API 开发教程 X402 价格说明

这篇文档说明用 X402 调用 Ace Data Cloud API 时，价格是怎么定的，以及它和平台普通 Credits（额度）价格是什么关系。

核心结论

X402 不是一套独立的价格体系。每个 API 调用本来就有一个以 Credits 计的成本，这是所有计费方式（API Token 扣额度、账户余额、X402 链上支付）共用的唯一价格来源。X402 只是把这个 Credits 成本按一个固定汇率换算成链上 USDC：

1 Credit = 0.095215 USDC（即 95215 atomic USDC，USDC 为 6 位小数）

也就是说：

这个汇率不是凭空设定的，它等于平台 最优档（大额）Credits 单价。所以用 X402 付费 ≈ 用最优档单价买 Credits 再消费，没有 X402 溢价，反而直接享受最便宜的 Credits 单价，且无需预充值、无需 API Token。

计价基准

项目 值 说明
换算汇率 1 Credit = 0.095215 USDC 平台统一设定的固定汇率
atomic 单位 1 Credit = 95215 atomic USDC USDC 6 位小数，atomic = floor(credits × 0.095215 × 1e6)
最小扣费 1 atomic USDC（$0.000001） 即使成本极低，也至少结算 1 atomic
计价币种 USDC Base / SKALE 为 ERC-20 USDC，Solana 为 SPL USDC

换算由平台统一完成，金额对所有支持的网络（Base、SKALE、Solana）一致；不同网络只是资产合约和签名方式不同，价格相同。

和普通 Credits 单价的关系

平台 Credits 按「用量越大、单价越低」的阶梯出售。X402 按调用计费时，统一使用阶梯最优档单价：

购买/支付方式 大致 Credits 单价 说明
入门 / 小额充值 约 $0.13 / Credit 列表价，小额买 Credits 时的单价
大额 / 最优档 $0.095215 / Credit 最便宜的单价
X402 按调用付费 $0.095215 / Credit 始终等于最优档单价

换句话说，X402 把每次调用都按最优档 Credits 单价结算。小额充值买 Credits 的用户，单价会高于 X402；大额充值用户的单价才和 X402 一致。

两种价格形态：exact 与 upto

X402 的 402 响应里，maxAmountRequired 就是你要签名的金额。它有两种形态：

  • exact（固定价）：请求处理前价格就能确定（图片、视频、音乐、搜索、订单支付）。maxAmountRequired = 该调用 Credits 成本 × 0.095215，即该次调用的应付金额（与成功响应里的 cost.amount 一致）。
  • upto（用量后置计量）：聊天补全这类「响应结束才知道用了多少 token」的接口。maxAmountRequired 是一个上限（按该模型设定的上限额度换算：上限 Credits × 0.095215），实际按真实用量结算，且不超过上限。

关于这两种 scheme 的协议细节见 exact 与 upto 计费方案。下面只讲价格。

Chat（聊天补全）价格

聊天补全是 upto 计量：按 prompt / completion token 实际用量计费。每百万（1M）token 的价格 = 该模型每 1M token 的 Credits 成本 × 0.095215，与平台公开模型目录（/api/v1/models?with_pricing=true）展示的 USD 单价完全一致。

  • 第一次请求没有携带 Authorization 或 X-Payment，所以返回 HTTP 402，不会产生扣费。
  • accepts 是本次请求唯一可信的签名依据，包含可选网络、scheme、金额上限、收款地址和资产地址。

字段含义：

字段 说明
scheme 支付方案。exact 表示固定金额，upto 表示授权上限、按实际用量结算。
network 支付网络，例如 base、skale、solana。
maxAmountRequired 最大支付金额，单位是 USDC atomic units，95215 表示 0.095215 USDC。
payTo 收款地址。
asset USDC 合约地址或 Solana mint 地址。
extra 签名需要的链 ID、EIP-712 domain、Permit2 地址等扩展信息。

技术教程 Claude 对接 ShortURL MCP

📘 总览指南：在 Claude 里用全部 11 个 AceData Cloud MCP →

长链丢给微信、Twitter、海报、PPT，永远是个麻烦事——又丑又截断。挂上 AceData Cloud 的 ShortURL MCP，Claude 在聊天框里一句话就能给你回一个 surl.id/xxx 的短链，复制即用。

本文是一篇亲手在 Claude.ai 网页版跑通的实测记录：从添加 Connector、OAuth 授权，到一句 prompt 把一条长 URL 缩成 https://surl.id/1uNbTadEob，全程截图，复制即可复现。

什么是 MCP？

MCP（Model Context Protocol）是 Anthropic 推出的开放协议，让 Claude 这类模型可以通过标准接口调用外部工具。简单理解：给 Claude 装上插件能力。

AceData Cloud 把 ShortURL 的核心能力封装成了一个远程 MCP Server，挂上去之后 Claude 就能直接：

  • 单个短链生成（shorturl_create）
  • 批量短链生成（shorturl_batch_create）
  • 查询 API 信息 / 配额（shorturl_get_api_info）
  • 查阅使用指南（shorturl_get_usage_guide）

整套接入只需一个 URL，剩下交给 Claude。

在 Claude.ai 网页版添加 ShortURL MCP

💡 Claude.ai 网页版接入是 OAuth 直连，不需要复制 API Token。你只需要一个能登录 AceData Cloud 的账号（GitHub / Google / 微信均可）。

1. 打开 Connectors 设置

登录 https://claude.ai，点左下角头像 → Settings → Customize → Connectors，或直接访问 https://claude.ai/customize/connectors。

⚠️ 旧入口 /settings/connectors 已经迁移到 /customize/connectors，访问旧地址会看到一条 “Connectors have moved to Customize” 的提示。

2. 添加 ShortURL MCP Server

点击右上角加号 → Add custom connector，在弹出对话框里填：

字段 值
Name ShortURL（随便取）
URL https://shorturl.mcp.acedata.cloud/mcp

点 Add，Claude 会自动发现 Server 暴露的全部工具。

3. OAuth 授权

点 Connect → 跳转到 AceData Cloud OAuth 授权页 → 用账号登录、点 Allow 即可。如果还没注册，先去 https://platform.acedata.cloud 注册一下，注册即送免费额度，足够你跑完本文的 demo。

4. 检查工具列表 & 把工具切到 Always allow

授权完成后回到 Claude.ai，点开 ShortURL Connector，能看到 4 个工具：shorturl_create、shorturl_batch_create、shorturl_get_api_info、shorturl_get_usage_guide。

⚙️ 强烈建议把右上角的 Needs approval 下拉切换到 Always allow，否则每次工具调用都会弹确认框。一次缩短只触发 1 次工具调用，但用多了还是建议直接 Always allow。

常见问题

Q：为什么我点了 Always allow，新工具还是要确认？
A：Always allow 是按工具粒度记忆的。create、batch_create、get_api_info、get_usage_guide 是 4 个不同工具，第一次各确认一次即可。

Q：余额不够怎么办？
A：错误信息里会带 insufficient balance。去 https://platform.acedata.cloud/console/coin 充值，支持支付宝 / 微信 / Stripe / USDC。短链单次消耗极低，新账号免费额度够你缩几百次。

Q：和 bit.ly / t.cn 比有什么优势？
A：surl.id 是 AceData Cloud 自营域名，不会突然被墙、不会强插广告页；配合 MCP 让 Claude 自动批量缩链，比手动一条条粘 bit.ly 强太多；可以和其他 MCP 串成工作流（出图 → 上传 → 缩链 → 发文）。

Q：Claude Desktop 和 Claude Code 也能这么用吗？
A：能。详见 Claude Desktop + ShortURL MCP 和 Claude Code + ShortURL MCP。

API 开发教程 Nano Banana Images API 对接说明

本文介绍 Nano Banana Images API 的对接与使用。该接口支持两种能力：图像生成（generate） 与 图像编辑（edit）。

申请流程

要使用 Nano Banana Images API，首先到 Ace Data Cloud 控制台 获取您的 API Token，留作备用。

如果你尚未登录或注册，会自动跳转到登录页面邀请你注册和登录，完成后会自动返回当前页面。

一个 API Token 即可调用平台所有服务，无需为每个服务单独申请。 首次申请会赠送免费额度，可免费体验；额度不足时可在 控制台 充值通用余额。

📘 完整文档：Nano Banana Images API →

接口概览

  • Base URL：https://api.acedata.cloud
  • Endpoint：POST /nano-banana/images
  • 认证方式：HTTP Header 中携带 authorization: Bearer {token}

  • 请求头：

    • accept: application/json
    • content-type: application/json

  • 动作（action）：

    • generate：根据文本提示词生成图片

    • nano-banana-pro：基于 Gemini 3 Pro Image Preview，最高质量

  • 异步回调：可选，通过 callback_url 接收任务完成通知与结果
