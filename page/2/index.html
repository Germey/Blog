产品推广 OpenCode MCP Overview

OpenCode 本身擅长读代码、改文件、跑命令、解释报错；MCP 则让它在同一个终端会话里继续调用外部工具。AceData Cloud 把音乐、图片、视频、搜索、短链这些能力做成托管式远程 MCP Server，你不需要本地启动服务，也不需要为每个工具维护不同账号。

这篇是总览入口，重点帮你判断：OpenCode MCP 能做什么、该先接哪个服务、每个服务的详细教程在哪里。具体配置写法、Token 获取、global / project 两种位置的差异，放在下面每篇单服务文章里展开。

如果你的 OpenCode 还不能正常调用模型，请先看 OpenCode 终端配置教程。如果你想直接对接 Claude Code 而不是 OpenCode，请看 Claude Code MCP 总览。

为什么要在 OpenCode 里接 MCP

OpenCode 是完全开源的终端编程 Agent，它的强项是”原生终端体验 + 任意模型可换”。接入 MCP 之后，它可以把”写完代码 → 补 README 配图 → 做发布素材 → 查参考资料 → 压短链分享”这一长串本来需要切换浏览器和工具的工作，全部在终端会话里完成。

和 IDE 类工具相比，OpenCode 更适合：

  • SSH 远程开发场景（无 GUI 也能完整使用 MCP）
  • 用脚本/CI 把 MCP 调用拼接进自动化流程
  • 喜欢键盘流、不希望被插件挡住屏幕的工程师

配置在哪里写

OpenCode 用同一份 opencode.json 描述供应商和 MCP。两种位置只是优先级不同：

  • 全局：~/.config/opencode/opencode.json，所有项目共用，适合”我自己的机器，一次配好”。
  • 项目级：当前项目根目录下的 opencode.json，仅对该项目生效，会覆盖全局，适合团队共享或单仓库定制。

无论放在哪里，结构都是相同的，每个服务一段 mcp.<name> 配置：

字段说明：

字段 说明
type 固定为 remote，代表远程托管 MCP Server
url AceData 提供的对应服务地址
enabled 是否启用，临时关闭某个 MCP 时改成 false 即可
oauth 必须显式设为 false。AceData MCP 走 Bearer Token 鉴权，不走 OAuth；不写这一行时 OpenCode 会把 401 当成 OAuth 挑战，最终 opencode mcp list 报 SSE error: Non-200 status code (401)
headers.Authorization Bearer <token>，推荐用 {env:ACEDATACLOUD_API_KEY} 占位符引用环境变量

{env:ACEDATACLOUD_API_KEY} 让 OpenCode 在启动时从环境变量里读取 Token，避免把真实 Token 直接写入文件。建议在 ~/.zshrc / ~/.bashrc 里 export ACEDATACLOUD_API_KEY="你的Token"。如果 Token 写在没有 export 的 .env 文件里，需要用 set -a && source .env && set +a 把它导出到 OpenCode 进程，否则 opencode debug config 会显示 "Authorization": "Bearer "（占位符未解析）。

配置示例里不会写完整 Token，只会用类似 357265014ad145fc8ab2.... 的形式展示。真实 Token 请只保存在你本机配置或私有环境变量中。

API 开发教程 上传文件到 AceDataCloud 平台 CDN

把本地文件上传到 AceDataCloud 平台 CDN，获得一个永久可访问的 URL（https://cdn.acedata.cloud/xxxxxx.png 风格），可立即在其他业务接口（如 Midjourney、Suno、Flux）的 image_url、reference_url 等参数中引用。

适用场景：

  • 在调用 Midjourney /midjourney/imagine 前先把垫图上传到 CDN 拿到 URL。
  • 把上一步生成的视频缓存起来用作下一步的输入。
  • 临时托管用户上传的素材，避免自建 S3。

ℹ️ 本接口属于 AceDataCloud 平台管理 API，统一前缀 https://platform.acedata.cloud/api/v1/。

项 内容
方法 POST
URL https://platform.acedata.cloud/api/v1/files/
鉴权 ✅ 需要账户令牌
Content-Type multipart/form-data（不是 JSON）

鉴权说明（如何获取账户令牌）

请求头：

获取方式：登录 AceDataCloud 平台 → Account Token 控制台 → 点「创建」按钮。详见管理 AceDataCloud 平台账户令牌。

请求体（multipart/form-data）

字段 类型 必填 说明
file file ✅ 二进制文件流。单文件上限默认 50 MB
filename string 否 自定义文件名；不传则用 multipart 自带的文件名
metadata string 否 JSON 字符串，挂在文件上的元数据（注意是字符串不是对象，因为 multipart 不支持原生嵌套 JSON）

响应示例（HTTP 201）

直接把 url 字段拿去其他业务接口里用就行——CDN 永久公开可访问，无需鉴权。

字段 类型 说明

错误处理

HTTP code 含义
400 invalid 没附 file 字段、字段名不对、metadata 不是合法 JSON 字符串
401 not_authenticated 缺少账户令牌
413 payload_too_large 文件超过 50 MB（默认）

实用提示

  • 上传图片做垫图：Midjourney image_urls、Flux image_url、Veo reference_url 都接受 CDN URL，不接受 base64。先用本接口拿 URL 是常规操作。
  • url 是永久 URL——目前没有自动清理策略，但敏感数据请勿上传。如有需求请联系客服设短期 TTL。
  • metadata 字段：multipart/form-data 不支持原生嵌套 JSON，传字符串：”{\"campaign\":\"summer\"}“，平台会反序列化。
  • 大批量上传：推荐 2-4 个并发，单连接即可达到 ~10 MB/s。

整体流程是：客户端发起请求的时候，额外指定一个 callback_url 字段，客户端发起 API 请求之后，API 会立马返回一个结果，包含一个 task_id 的字段信息，代表当前的任务 ID。当任务完成之后，生成视频的结果会通过 POST JSON 的形式发送到客户端指定的 callback_url，其中也包括了 task_id 字段，这样任务结果就可以通过 ID 关联起来了。

1.Luma AI是什么

Luma 是一个专业高质量的视频生成平台，用户只需上传图片素材，即可根据不同风格和效果自动生成高质量视频。Luma 推出的 Dream Machine视频生成器，基于 DiT 视频生成架构，其核心科技是 NeRF，即神经辐射场技术，能在 120 秒内产出包含 120 帧的高品质视频，从而实现了创意的快速迭代。这款工具在电影制作、建筑可视化以及虚拟现实场景构建中有广泛应用，帮助创作者快速生成逼真的虚拟环境。

应用领域：电影制作、建筑可视化、虚拟现实。

2.Luma 功能

  1. 关键帧功能允许用户设定视频的起始和结束关键帧。通过这一功能，创作者们可以实现前所未有的叙事控制和创意表达，让创作者能够更好地掌控视频的节奏和情感变化，从而在AI视频领域开创更多可能性。例如，在制作一个短片时，创作者可以设定一个充满悬念的开头和一个高潮迭起的结尾，通过关键帧功能自然地过渡，从而吸引观众的注意力并保持他们的兴趣。

3.Dream Machine 的优势

Luma AI 支持从文本或图片生成视频，能够快速产出 5 秒的高质量视频。

优势一：高效快速，120 秒内即可生成 120 帧视频。

得益于帧率的显著提升，视频流畅度更高。

优势二：动作逼真且连续。

能够实现电影级的镜头运动效果，在镜头移动时保持画面的流畅和自然。

优势三：角色一致性高，能模拟物理世界。

Dream Machine 的新结构使其能够理解人、动物和物体在物理世界中的交互。对于镜头中的人物，不同角度的变换也能更好地保持形象的一致性。

优势四：运镜自然，与场景情感完美契合。

能够生成流畅的动态镜头，显著改善视觉体验，增强 AI 视频的故事表现力，打造影视级的观赏体验。

技术教程 独立开发者的第二收入源：用你的代码能力撬动分销收入

作为独立开发者，你可能有过这样的困境：手里的技术能力很强，但变现方式有限——要么接外包、要么做独立产品、要么打工。

今天介绍一条很多开发者还没意识到的路径：用你的代码能力做分销。

不是让你去发传单、拉群卖货。而是——你做的每一个工具、开源项目、Demo 应用，都可以成为你的分销渠道。 当你的用户通过你的项目接入了 Ace Data Cloud 的 API，你就能持续获得佣金。

一、开发者做分销的独特优势

你做的工具就是你的分销渠道

一个普通推广者要写文案、做海报、群发消息。但你是开发者——你做的产品本身就是最好的推广。

用户用你的工具 → 工具调用 Ace Data Cloud API → 用户需要注册并充值 → 你获得佣金

这个链条天然成立，不需要任何”推销”行为。

开源项目 = 永不停歇的推广机器

一个有 star 的 GitHub 项目，每天都有人来看 README、clone 代码、试用。只要你在 README 中引导用户注册时使用你的邀请链接，每个尝试你项目的人都可能成为你的下线。

技术壁垒 = 竞争护城河

非开发者做分销，大家都在同一条赛道上竞争——写文章、发帖、做视频。而你可以用代码做出别人做不了的东西：SDK、CLI 工具、VS Code 插件、Telegram Bot……这些是只有开发者才能构建的分销渠道。

二、佣金回顾

等级 累计邀请用户消费额 佣金比例
Level 1 $0+ 10%
Level 2 $1,000+ 15%
Level 3 $5,000+ 20%

用户永久绑定，跨平台（platform.acedata.cloud、studio.acedata.cloud）消费都算你的佣金。还有二级分销——你发展的推广者再邀请的用户，消费时你也能拿 20% 佣金分成。

三、7 个具体的项目方向

方向一：Python / Node.js SDK 封装

把 Ace Data Cloud 的 API 封装成好用的 SDK 库，发布到 PyPI / npm。

变现逻辑： 用户要用你的 SDK → 需要 API Key → 通过你的引导注册 Ace Data Cloud → 绑定为你的下线

在 README 中写：”获取 API Key → 注册 Ace Data Cloud“，附带你的邀请链接。

方向二：CLI 命令行工具

开发者喜欢命令行。做一个好用的 CLI 工具：

Ace Data Cloud 平台上已经有多个开源 CLI 项目可以参考：Suno CLI、Luma CLI、Sora CLI、Veo CLI 等。你也可以 fork 这些项目，做二次开发或做整合版。

方向三：VS Code 扩展

VS Code 是全球最流行的编辑器，月活用户超过 3000 万。做一个调用 AI API 的扩展：

  • AI 代码助手：选中代码 → 右键 → “用 AI 解释 / 重构 / 补全”
  • AI 翻译：选中文本 → 翻译成多种语言
  • AI 图片生成：输入 prompt → 在编辑器中预览生成的图片

用户安装你的扩展 → 配置 API Key → 注册 Ace Data Cloud → 成为你的下线。

方向四：Telegram / Discord Bot

做一个功能丰富的 AI Bot：

  • /chat 问答对话（ChatGPT / DeepSeek / Claude）
  • /image 生成图片（Midjourney / Flux）
  • /music 生成音乐（Suno）
  • /video 生成视频（Luma / Sora）

Bot 可以通过代理你的 API Key 来运行（用户免注册但你控制额度），也可以引导用户注册自己的账号使用。后者更适合分销——每个用户注册后永久绑定。

商业模式： Bot 在群里运行 → 用户试用 → 深度用户引导去注册 → 佣金到手。

方向五：开源 AI 应用模板

做一个可以快速部署的 AI 应用模板：

  • AI 壁纸生成器：用户输入描述 → 自动生成手机/电脑壁纸
  • AI 名片制作器：上传照片 → AI 生成创意名片
  • AI 音乐工作站：在线创作 AI 音乐的 Web 应用
  • AI 短视频工坊：输入脚本 → 自动生成短视频

这些模板可以用 Next.js / Vue / React 快速搭建。关键是在模板的注册流程中引导用户通过你的链接注册 Ace Data Cloud。

方向六：用 Nexior 搭建自己的 AI 平台

Nexior 是 Ace Data Cloud 的开源前端应用，你可以直接 fork 它，搭建一个你自己品牌的 AI 平台。

  • 自定义域名、Logo、品牌色
  • 内置 AI 聊天、AI 绘画、AI 音乐、AI 视频全部功能
  • 用户系统和 Ace Data Cloud 打通

核心价值： 通过你的平台注册的每个用户都是你的下线。你不需要自己运维 AI 服务——Nexior 直接调用 Ace Data Cloud 的 API，你只管运营和获客。

方向七：Chrome 扩展

做一个浏览器插件：

  • 右键选中文本 → AI 翻译/总结/改写
  • 截图 → AI 识别并分析
  • 看到好的图片 → AI 生成类似的变体

Chrome Web Store 的曝光量不可小觑，一个好用的 AI 扩展很容易获得数千安装量。

  • 进入 Level 2 后：$360/月

五、实操建议

1. 选择你最擅长的技术栈

不要为了分销去学新技术。用你最熟悉的语言和框架，做你最擅长的东西。

2. 先解决一个真实问题

不要为了分销而做项目。想一想：你自己在用 AI API 时，有什么不方便的地方？把它做成工具，这就是最好的项目方向。

3. README 要好好写

GitHub 项目的转化全靠 README。确保：

  • 开头就说清楚这个工具是做什么的
  • 有 GIF 或截图展示效果
  • 安装/使用步骤清晰
  • 获取 API Key 的引导放在显眼位置（自然嵌入邀请链接）

4. 持续维护

一个活跃维护的项目比一个弃坑的项目有用得多。定期更新、回复 Issue、接受 PR——这些都会吸引更多用户。

5. 多渠道推广你的项目

项目做完了，去这些地方发帖：

  • Hacker News / Reddit / Product Hunt
  • 知乎 / V2EX（项目分享）
  • Twitter / X（开发者社区）

一次推广可能带来几百个 star 和几十个注册用户。
